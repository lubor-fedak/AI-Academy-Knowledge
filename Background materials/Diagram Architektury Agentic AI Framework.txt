3) Komponenty — čo robia a návrhové rozhodnutia
Orchestrator / AI Core

Zodpovednosť: Riadi plánovanie úloh, rozhoduje ktoré agenty/úlohy spustiť, koordinuje multi-agentné scénare a rozhoduje o human-in-the-loop eskaláciách. Tiež komunikuje s model routerom pre výber LLM/strategie.

Kľúčové vlastnosti: stateful runtime, retry/persistence, orchestration policies, pluggable workflow engine.

Odporúčanie technológií: Kubernetes + Knative / serverless runtime; workflow: Temporal/Argo/Zeebe; orchestrator logika v microservice (Python/Go).

Dokument: Orchestrátor = runtime intelligence, agent orchestration vs workflow rozdiel.

5) Bezpečnosť, compliance a data-sovereignty

Zero-trust: každé volanie autentifikovať a autorizovať; RBAC + scopes. (Kyndryl zdôrazňuje zero-trust) 

Agentic AI Framework

Policy enforcement: OPA/PDPS pre pravidlá, napr. ktoré modely použiť s citlivými dátami, ktoré akcie sú povolené.

Audit: plné záznamy: prompt, model ver., odpoveď, akcie, výsledky, kto schválil.

Encryption & KMS: encrypt-at-rest a encrypt-in-transit. Separácia kľúčov podľa tenantov/region.

Data sovereignty: možnosť regionálneho vrstvenia (lokálne vector DB alebo souverain cloud).

Safety: content-filtering, guardrails, red-team testing pred nasadením agentov (safety-tests v CI).

6) Model routing a optimalizácia nákladov

Model Router rozhoduje podľa: latencie, cena/požiadavka, citlivosti dát (sú to PII?), SLA. Môže použiť: preference → cena/perf skóring → fallback chain.

Cache & batching: cache výsledkov, batch requesty pre embeddings, rate-limiting.

Fallbacks: ak externé LLM zlyhá, fallback na lokálny model alebo heuristiku.

7) Príklad multi-agent orchestration (konkrétny scenár)

Scenár: „Zisti root cause incidentu, navrhni remediation, vytvor ticket v ServiceNow a pošli summary vedúcemu.“

Kroky:

Orchestrator aktivuje „Investigation Agent“ → vykoná RAG lookup do logs vector DB a konfiguračnej knowledge.

Investigation Agent použije Root-Cause Agent (specializovaný) → vráti hypothesisy.

Remediation Agent (beží súbežne) simuluje nápravné kroky v sandboxe.

Orchestrator vyhodnotí confidence; ak nad threshold → exekvuje ToolSandbox: vytvor ServiceNow ticket. Inak → HITL na review.

Všetky kroky sa auditujú a výsledky idú do long-term memory pre learning.
(Orchestrácia a multi-agentná spolupráca sú explicitne podporované v dizajne dokumentu.)

8) Rýchle odporúčania pre implementačný plán (MVP -> Production)

MVP: Orchestrator + Agent Registry + jednoduchý Agent Exec + Vector DB + jednoduchý model router (1 externý LLM + 1 lokálny). Low-code builder minimalne.

Stredná fáza: pridaj Policy Engine, sandboxed Tool execution, short + long memory, konektory.

Prod: multitenancy, full governance (audit, compliance reports), cost governance, CI/CD, monitoring, autoscaling a model routing.

Testy: Safety tests + red-team + performance/chaos tests.
