# Hackathon Presentation Template

## Slide Structure (10 slides maximum)

---

### Slide 1: Title
- Team name
- Solution name
- One-line tagline

---

### Slide 2: The Problem
- What pain are we solving?
- Who has this pain?
- What does it cost?

---

### Slide 3: Our Solution
- One-sentence description
- Key differentiator
- Screenshot or mockup

---

### Slide 4: Demo
[LIVE DEMO - 4 minutes]

Show:
1. Happy path (works perfectly)
2. Edge case handling
3. Error recovery

---

### Slide 5: Architecture
- Simple diagram (max 6 boxes)
- Key technology choices
- Why this approach

---

### Slide 6: Security
- Threats considered
- Mitigations implemented
- What we'd add with more time

---

### Slide 7: Quality
- How we know it works
- Metrics/evaluation
- Test coverage

---

### Slide 8: Business Case
- ROI calculation
- Time to value
- Scalability path

---

### Slide 9: Roadmap
- Phase 2 features
- Production requirements
- Timeline estimate

---

### Slide 10: The Ask
- What we need to proceed
- Call to action
- Thank you

---

## Presentation Tips

### DO:
- Start with a hook
- Show, don't tell
- Have backup plan if demo fails
- Practice timing (strict 10 min)
- Make eye contact

### DON'T:
- Read slides verbatim
- Go over time
- Apologize for what you didn't build
- Use jargon without explaining
- Rush the demo
