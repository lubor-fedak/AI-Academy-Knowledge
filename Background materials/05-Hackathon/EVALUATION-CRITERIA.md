# Hackathon Evaluation Criteria

## Scoring Breakdown

| Criterion | Weight | Max Points |
|-----------|--------|------------|
| Business Value | 25% | 25 |
| Technical Excellence | 25% | 25 |
| Innovation | 20% | 20 |
| Presentation | 15% | 15 |
| Feasibility | 15% | 15 |
| **Total** | **100%** | **100** |

## Detailed Rubrics

### Business Value (25 points)

| Score | Description |
|-------|-------------|
| 21-25 | Clearly solves stated problem, quantified ROI, addresses stakeholder needs |
| 16-20 | Solves problem well, some value quantification |
| 11-15 | Partially addresses problem, value unclear |
| 6-10 | Weak connection to business need |
| 0-5 | No clear business value |

### Technical Excellence (25 points)

| Score | Description |
|-------|-------------|
| 21-25 | Clean architecture, proper error handling, scalable design |
| 16-20 | Good implementation, minor issues |
| 11-15 | Works but has significant technical debt |
| 6-10 | Barely functional |
| 0-5 | Does not work |

### Innovation (20 points)

| Score | Description |
|-------|-------------|
| 17-20 | Creative use of agentic patterns, novel approach |
| 13-16 | Some innovative elements |
| 9-12 | Standard implementation |
| 5-8 | Basic approach |
| 0-4 | No innovation |

### Presentation (15 points)

| Score | Description |
|-------|-------------|
| 13-15 | Clear, engaging, well-structured demo |
| 10-12 | Good presentation, minor issues |
| 7-9 | Adequate but unclear in parts |
| 4-6 | Confusing or poorly organized |
| 0-3 | Unable to present effectively |

### Feasibility (15 points)

| Score | Description |
|-------|-------------|
| 13-15 | Could deploy to production with minimal changes |
| 10-12 | Feasible with some additional work |
| 7-9 | Significant work needed for production |
| 4-6 | Major concerns about viability |
| 0-3 | Not feasible for production |
